Nextcloud Desktop Client on Ubuntu - Step-by-Step Guide

This tutorial summarizes the process of installing, cleaning, and using the Nextcloud desktop client on Ubuntu using the AppImage format.

1. Understanding the Setup

  • Nextcloud AppImage = Desktop client, only syncs files.
  • Nextcloud Snap = Full server (PHP, MySQL, Redis). Not needed if you only want client.
  • Always run the AppImage without sudo.

2. Cleaning Old Installations

If you had previous installations (AppImage or Snap), clean leftover processes and configs:

# Kill any running AppImage
pkill -f Nextcloud-4.0.6-x86_64.AppImage

# Remove old config folders
rm -rf ~/.config/Nextcloud
rm -rf ~/.local/share/Nextcloud

# If you previously installed Snap server
sudo snap stop nextcloud
sudo snap remove nextcloud
sudo rm -rf /var/snap/nextcloud /snap/nextcloud

# Fix permissions in home folder
sudo chown -R $USER:$USER ~/
Note: Always backup any important files before removing config folders.

3. Starting Fresh with the AppImage

  1. Make the AppImage executable: 
    chmod +x ~/Applications/Nextcloud-4.0.6-x86_64.AppImage
  2. Launch it: 
    ~/Applications/Nextcloud-4.0.6-x86_64.AppImage
  3. Connect to your university cloud URL via the browser login.
  4. Choose a clean local folder for syncing (e.g., ~/Nextcloud).

4. Handling “Folder Already in Use” Errors

If the client says a folder is already in use:

  1. Quit the client: pkill -f Nextcloud-4.0.6-x86_64.AppImage
  2. Check the folder for leftover state files: 
    ls -la ~/Nextcloud
# Look for .sync_config.db, .sync_*.log, .Nextcloud
  3. Backup and remove these files: 
    mkdir ~/Nextcloud_backup_old
mv ~/Nextcloud/.sync_config.db ~/Nextcloud/.sync_*.log ~/Nextcloud/.Nextcloud ~/Nextcloud_backup_old/
  4. Restart the AppImage and select the original folder.

5. Verifying Sync

  • Tray icon: ✅ green check = synced, 🔄 arrows = syncing
  • Test sync by creating a file: 
    touch ~/Nextcloud/test_sync.txt
  • Check that it appears in your university cloud web interface.

    • 6. Optional Settings

    • Enable autostart: Tray icon → ⚙ Settings → Launch on system startup
    • Keep AppImage in a permanent location (e.g., ~/Applications)
    • Merge old backup files if needed; the client will sync them automatically

    7. Summary

    • Clean old configs and leftover Snap server before installing the client.
    • Use only the AppImage for desktop sync.
    • Backup old folders if migration errors appear.
    • Set folder permissions properly to avoid “already in use” errors.
    • Tray icon shows sync status and allows autostart configuration.

    Now your Nextcloud desktop client is fully integrated with Ubuntu Files, syncing automatically with your university cloud.
